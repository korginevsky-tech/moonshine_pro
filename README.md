# 🥃 MOONSHINE CONTROLLER PRO v1.0

Полная система автоматизации ректификационной колонны на базе ESP32.

## ✨ Возможности

### 🎛️ **Управление**
- ✅ Старт/Стоп процесса с таймером
- ✅ Контроль нагревателя (реле)
- ✅ Контроль насоса охлаждения
- ✅ Управление клапаном царги
- ✅ Регулировка скорости отбора (0-100%)

### 💧 **Фракции отбора**
- 👑 **ГОЛОВА** - 60% скорости
- 💪 **ТЕЛО** - 100% скорости  
- 🔚 **ХВОСТЫ** - 30% скорости
- ⏹️ **СТОП** - 0% скорости

### 🌡️ **Контроль температуры**
- Два независимых датчика DS18B20
- PID регулятор для куба
- Автоматическая калибровка царги
- Определение этапов процесса

### 🚨 **Безопасность**
- Контроль перегрева (≥105°C) → автоматическое отключение
- Предупреждение при ≥100°C
- Контроль отключения датчика
- LED индикаторы статуса

### 📊 **Мониторинг**
- Веб-интерфейс в реальном времени
- График температур обоих датчиков
- История последних 1000 измерений
- Логирование всех событий
- Скачивание результатов в JSON

## 🔧 Требования

### Оборудование
- **ESP32** (микроконтроллер)
- **DS18B20** (×2 датчика температуры)
- **4-канальное реле** (230В)
- **PWM модуль** (для управления скоростью отбора)
- **Питание**: 5V для ESP32, 230V для реле

### Библиотеки Arduino
```
- OneWire
- DallasTemperature
- WiFi
- WebServer
- SPIFFS
- ArduinoJson
- Preferences
```

## 📦 Установка

### 1️⃣ Загрузка кода
```bash
# Откройте Arduino IDE
# File → Open → moonshine_pro.ino
# Установите плату: ESP32 Dev Module
# Выберите COM порт
# Нажмите Upload (→)
```

### 2️⃣ Загрузка файлов в SPIFFS
```bash
# Tools → ESP32 Sketch Data Upload
# Файлы из папки data/ загрузятся в SPIFFS
```

### 3️⃣ Конфигурация WiFi
Отредактируйте в коде:
```cpp
#define WIFI_SSID "your_ssid"
#define WIFI_PASSWORD "your_password"
```

### 4️⃣ GPIO пины
По умолчанию:
```cpp
#define TEMP_SENSOR_PIN 4      // Датчики 1-Wire
#define HEATER_RELAY_PIN 18    // Реле нагревателя
#define PUMP_RELAY_PIN 19      // Реле насоса
#define VALVE_RELAY_PIN 25     // Реле клапана
#define OUTLET_PWM_PIN 23      // PWM отбора
#define ALARM_LED_PIN 21       // LED тревога
#define STATUS_LED_PIN 22      // LED статус
```

## 🚀 Использование

### Первый запуск
1. Откройте Serial Monitor (115200 baud)
2. Найдите IP адрес в логах
3. Перейдите на `http://192.168.X.X` в браузер

### Инициализация датчиков
1. Вкладка **🌡️ Датчики**
2. Кнопка **🔍 СКАНИРОВАТЬ ДАТЧИКИ**
3. Выберите роли для каждого датчика:
   - 🔥 КУБ (обязательно)
   - 👑 ЦАРГА (обязательно)
   - ❄️ ХОЛОДИЛЬНИК (опционально)

### Запуск процесса
1. Вкладка **📊 Панель**
2. Установите целевую температуру куба
3. Нажмите **▶️ СТАРТ ПРОЦЕССА**
4. Система начнёт разогрев

### Контроль отбора
1. Вкладка **💧 Отбор**
2. Выбор режима: ГОЛОВА → ТЕЛО → ХВОСТЫ
3. Или ручное управление ползунком (0-100%)

### Мониторинг
1. Вкладка **📈 График** - визуализация обеих температур
2. Вкладка **📝 Логи** - все события и действия
3. Вкладка **🎛️ Управление** - ручное управление всеми компонентами

## 🔌 Подключение оборудования

### Датчик температуры DS18B20
```
DS18B20 (3 выводов):
  VCC (красный) → +5V
  GND (чёрный) → GND
  DQ (жёлтый)  → GPIO 4 (резистор 4.7kΩ к VCC)
```

### Реле (4-канальное)
```
Канал 1 (IN1) → GPIO 18 (Нагреватель)
Канал 2 (IN2) → GPIO 19 (Насос)
Канал 3 (IN3) → GPIO 25 (Клапан)
Все GND → общий GND
```

### PWM модуль отбора
```
PWM Signal → GPIO 23
GND → общий GND
```

## 🐛 Решение проблем

### ESP32 не подключается к WiFi
- Проверьте SSID и пароль в коде
- Убедитесь в наличии 2.4GHz сети (5GHz не поддерживается)

### Датчики не найдены
- Проверьте подключение DS18B20
- Убедитесь в подтяжном резисторе 4.7kΩ
- Используйте Serial Monitor для диагностики

### Веб-интерфейс не загружается
- Проверьте загрузку файлов через `ESP32 Sketch Data Upload`
- Убедитесь что SPIFFS инициализирован

### Температура не меняется
- Проверьте подключение реле нагревателя
- Убедитесь что нагреватель включен в сети
- Проверьте PID параметры
